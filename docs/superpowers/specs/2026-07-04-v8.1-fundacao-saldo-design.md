# V8.1 — Fundação do Design System SALDO — Design

**Data:** 2026-07-04
**Marco:** V8 — Layout e estética · **Sub-marco:** V8.1 (fundação + shell + dashboard, responsivo)
**Fonte da identidade:** `Saldo Brand Kit.dc.html` (na raiz do repo). Produto: **SALDO** · `seusaldo.ai`.

## 1. Objetivo

Rebrandar o CRM como **SALDO**: instalar a identidade visual do Brand Kit (cores, tipografia, logo, favicon), criar os primitivos de UI reutilizáveis, e aplicar tudo ao **shell + sidebar + dashboard** de forma **responsiva** (desktop e mobile). Essa fundação é reusada no rollout às demais telas (V8.2).

## 2. Fronteiras (escopo)

**Dentro:** tokens de cor/tipografia (Tailwind 4 `@theme`), fontes self-hosted (`next/font`), `LogoSaldo` + favicon, primitivos (`Card`, `Botao`, `Badge`, `PageHeader`, `StatCard`), sidebar SALDO com drawer mobile, shell (fundo creme), dashboard (home) reestilizado e responsivo.

**Fora (V8.2 e além):** demais telas (clientes, financeiro, NFS-e, atendimento, integrações, usuários) — só serão migradas no rollout; **modo whitelabel** (V9 — os tokens já ficam preparados como variáveis, mas sem troca dinâmica de marca aqui); dark mode (o app é light-only, como a marca); ilustrações/gráficos avançados.

**Restrição:** não quebrar comportamento/funcionalidade — é mudança **visual**. Acessibilidade atual (foco-visível WCAG, skip-link, `aria-current`) é preservada/adaptada às cores novas.

## 3. Identidade (do Brand Kit)

**Cores:** Verde Saldo `#0FA968` (primária), Verde claro `#3CD98A` (acento em fundo escuro), Tinta `#0E1512` (sidebar/superfícies escuras) e `#101614` (texto), Violeta IA `#7C5CFF` (dados/destaque), Creme `#F7F6F2` (fundo), Cinza texto `#5A6163`, Cinza claro `#8C938E`, Linha `#E7E4DC`, Negativo `#C0503F`. Escuros auxiliares: `#111B16`, borda escura `#232A27`, mono muted `#78827C`, texto sobre escuro `#B4BCB6`. Cards = branco `#fff`.

**Tipografia:** **Space Grotesk** (700/600/500) — títulos e **números/valores**; **IBM Plex Sans** (400/500/600) — texto/UI; **IBM Plex Mono** (400/500) — dados tabulares (R$, CNPJ, códigos, labels).

**Logo:** moeda (círculo verde) + seta pra cima (triângulo + haste, branco) + wordmark "Saldo" (Space Grotesk 700). Variantes: padrão (claro), sobre escuro, reverso mono (sobre verde), só-símbolo. Favicon/ícone = quadrado arredondado verde com o símbolo.

## 4. Tokens + fontes

### 4.1 Fontes (`src/app/layout.tsx`)
Trocar `Geist` por três famílias via `next/font/google` (self-hosted no build — sem CDN em runtime):
- `Space_Grotesk` → `--font-display` (weights 500/600/700).
- `IBM_Plex_Sans` → `--font-sans` (400/500/600, subsets latin + latin-ext p/ acentos).
- `IBM_Plex_Mono` → `--font-mono` (400/500).
As três variáveis vão na `<html className>`. `viewport.themeColor` → `#0E1512` (tinta).

### 4.2 Tokens (`src/app/globals.css`, Tailwind 4 `@theme`)
Declarar as cores como `--color-verde`, `--color-verde-claro`, `--color-tinta`, `--color-texto`, `--color-violeta`, `--color-creme`, `--color-cinza`, `--color-cinza-claro`, `--color-linha`, `--color-negativo`, `--color-tinta-2` (`#111B16`), `--color-borda-escura`, `--color-mono-muted`, `--color-texto-claro` → geram utilitárias `bg-verde`, `text-tinta`, `border-linha`, etc. Mapear `--font-display/sans/mono`. `body` → `background: creme; color: texto; font: sans`. Foco-visível → outline **verde** (`#0FA968`); sobre a sidebar escura → branco.

## 5. Logo e favicon

- **`src/components/marca/LogoSaldo.tsx`** — SVG do símbolo (moeda+seta) + wordmark. Props: `variante` (`"claro" | "escuro" | "mono" | "simbolo"`), `tamanho` (px do símbolo, wordmark proporcional). Puro/estático (sem estado).
- **Favicon:** substituir `src/app/icon.svg` pelo símbolo SALDO (quadrado arredondado verde + seta branca) — Next gera o favicon do `icon.svg`.

## 6. Primitivos (`src/components/ui/`)

Componentes pequenos, tipados, sem estado (salvo indicado), estilizados com os tokens:
- **`Card`** — `bg-white border border-linha rounded-2xl p-*` (raio ~16px). Props: `className`, `children`.
- **`Botao`** — variantes `primario` (bg-verde texto branco), `secundario` (borda-linha, texto tinta), `fantasma` (sem borda), `perigo` (negativo). Renderiza `<button>`; aceita `type`, `disabled`, etc. (compatível com Server Actions — não força `onClick`).
- **`Badge`** — pílula pequena; variantes `neutro`/`positivo` (verde)/`atencao`/`negativo`/`ia` (violeta).
- **`PageHeader`** — título (`font-display`), subtítulo opcional e `acoes` (slot à direita); responsivo (empilha no mobile).
- **`StatCard`** — `rotulo` (mono, cinza), `valor` (font-display grande), `variante` de cor do valor (`neutro`/`positivo` verde/`destaque` violeta/`negativo`). Helper puro **`corValorStat(variante)`** → classe de cor (testável).

## 7. Shell + Sidebar (responsivo)

### 7.1 Sidebar (`src/components/Sidebar.tsx`)
- **Desktop (`md+`):** coluna fixa `bg-tinta` (`#0E1512`), largura ~15rem, `LogoSaldo variante="escuro"` no topo, itens de nav (texto `texto-claro`; hover `bg-tinta-2`; **ativo = `bg-verde text-white`**, mantendo `aria-current`), "Sair" no rodapé. Nome do usuário em mono muted.
- **Mobile (`<md`):** vira **barra superior** `bg-tinta` com a logo + botão **hambúrguer**; ao tocar, abre um **drawer** (overlay) com a mesma navegação. Estado `aberto` via `useState` (o componente já é client). Fecha ao navegar (clicar num item) e no overlay.
- Preserva a lógica de itens por papel (inclui "Atendimento", "Financeiro" etc.).

### 7.2 Shell (`src/app/(app)/layout.tsx`)
- `main` → fundo **creme**, padding responsivo (`p-4 md:p-6`). Skip-link preservado (cores ajustadas). Estrutura flex: sidebar (coluna no desktop / topo no mobile) + conteúdo.

## 8. Dashboard (home) — vitrine

`src/app/(app)/page.tsx` reestilizado com os primitivos, **mantendo os dados e a RLS atuais** (`dashboard_resumo`, recentes):
- **`PageHeader`** com saudação ("Início" → manter, com subtítulo tipo "N clientes ativos") + ação "Novo cliente" (`Botao primario`).
- Os cards de resumo viram **`StatCard`** (total/ativos/inativos; ativos em verde). "Por regime" e "recentes" em **`Card`** retematizados.
- Grid responsivo (`grid-cols-1 sm:grid-cols-2 lg:grid-cols-3`). Números em `font-display`.
- `CardResumo` legado: manter, mas a home passa a usar `StatCard` (o `CardResumo` só é migrado onde aparecer, na V8.2).

## 9. Testes e verificação
- **Unit:** `corValorStat(variante)` (mapa variante→classe) — pequeno e determinístico; e `LogoSaldo` renderiza o `<svg>` para cada variante (smoke test com testing-library, se já configurado; senão, só o helper puro).
- **Verificação:** `npm run lint && npm run typecheck && npm run build` (verde) + **checagem visual** no desktop e no mobile (viewport estreito): sidebar/drawer, dashboard, fontes carregando, cores corretas, foco-visível.
- Não há mudança de banco (sem migration).

## 10. Riscos e observações
- **Fontes no build:** `next/font/google` baixa e self-hosta no build (funciona no EasyPanel). Garantir subsets com `latin-ext` (acentos pt-BR).
- **Regressão visual:** só a home/shell/sidebar mudam de fato; as demais telas continuam com o estilo antigo até a V8.2 — pode haver leve inconsistência transitória (esperado; documentar).
- **Contraste/acessibilidade:** validar verde `#0FA968` como fundo de texto branco (item ativo/botão) — o kit usa esse padrão; manter foco-visível e `aria-current`.
- **Whitelabel (V9):** os tokens ficam como variáveis CSS, prontos para futura troca de marca; nesta fatia, marca fixa = SALDO.
- **Nome do produto:** títulos/`metadata` podem passar a "SALDO" — manter "CRM Contábil" onde for texto funcional; o rebrand de copy amplo fica para o rollout/decisão do usuário (nesta fatia: logo + título do app).
