# V8.2b — Rollout SALDO: Financeiro — Design

**Data:** 2026-07-04
**Marco:** V8 — Layout e estética · **Sub-marco:** V8.2b (2ª fatia do rollout)
**Depende de:** V8.1 (tokens/primitivos base), V8.2a (guia `docs/design/saldo-ui.md`, primitivos ampliados, helpers, mapa de tokens).

## 1. Objetivo

Aplicar a linguagem de design SALDO às **10 telas/componentes do módulo Financeiro**, com a mesma qualidade da fatia de Clientes, reusando os primitivos e o mapa de tokens já aprovados. **Só apresentação** — cálculos, actions, RLS e comportamento inalterados.

## 2. Fronteiras (escopo)

**Dentro:** dashboard financeiro, contas a receber, contas a pagar, régua de cobrança, o CRUD genérico de cadastros (contas bancárias / plano de contas / centros de custo / fornecedores / serviços) e os hubs/pages correspondentes. Um helper novo `badgeStatusTitulo`.

**Arquivos:** `src/app/(app)/financeiro/{dashboard,contas-a-receber,contas-a-pagar,regua-cobranca,cadastros}/page.tsx`, `src/app/(app)/financeiro/regua-cobranca/Regua.tsx`, `src/components/financeiro/{DashboardFinanceiro,ContasReceber,ContasPagar,CadastroCrud}.tsx`.

**Fora:** NFS-e (V8.2c); atendimento/integrações/resto (V8.2d). Sem mudança de banco/lógica.

## 3. Linguagem (reuso da V8.2a)

Aplicar o guia `docs/design/saldo-ui.md` e o **mapa de tokens** (Global Constraints do plano V8.2a): `slate-*`→tokens SALDO, botões primários→`Botao`/`verde`, sucesso→`verde/10`, erro→`negativo/10`, cards→`Card`/`Painel`, **valores monetários em `font-mono`**, títulos em `font-display`. Regra invariável: **re-skin, nunca refuncionalizar** (preservar `name`/`value`/`onChange`/actions/`aria`).

## 4. Toques específicos do Financeiro

- **Semântica de cor:** saldo/recebido/positivo = **verde**; despesa/vencido/erro = **negativo**; a receber/previsão/dados = **violeta** ou neutro; atenção (parcial) = **âmbar**. A cor semântica é separada do acento — indica estado, não decora.
- **`badgeStatusTitulo(status: string): variante`** (novo helper puro em `src/lib/ui/apresentacao.ts`) → variante de `Badge`: `ABERTO`→`neutro`, `BAIXADO`→`positivo`, `BAIXADO_PARCIAL`→`atencao`, `CANCELADO`→`neutro`, `VENCIDO`→`negativo`. Rótulos legíveis via `LABEL_STATUS` já existente.
- **Valores:** `formatarMoeda(...)` em `font-mono`, alinhados à direita nas tabelas; `tabular-nums`.

## 5. Telas

- **`DashboardFinanceiro`:** cards → `StatCard` (saldo real/MRR/recebido em `positivo`; saídas/a pagar neutros; inadimplência com % ; obrigações/violeta). As **barras** de aging e de fluxo passam a usar `bg-verde` (recebido/positivo) e `bg-linha`/`bg-violeta` (demais); seções (aging, fluxo, maiores devedores, receita×despesa) em `Card`/`Painel`, valores em mono. Manter o aviso/legendas atuais.
- **`ContasReceber` / `ContasPagar`:** listas de títulos por competência em `Painel` no padrão da lista de clientes (linha com hover, valores mono à direita, `Badge` via `badgeStatusTitulo`, saldo destacado). Formulários de **baixa/pagamento/lançamento** com `Campo`/`Input`/`Select`/`Botao`; ações (Baixar, Estornar, Cobrar via WhatsApp, Gerar) como `Botao` (variantes apropriadas — estorno/excluir = `perigo`/`fantasma`). Toggle de automação re-skinnado.
- **`Regua` + `regua-cobranca/page`:** toggle "ativa" e "Processar agora" como `Botao`; CRUD de etapas com `Campo`/`Input` em `Painel`; histórico em `Painel`/tabela.
- **`CadastroCrud` (genérico):** re-skin com `Painel` (lista) + `Campo`/`Input`/`Select` (form) + `Botao` (salvar/voltar/excluir). Como é usado por todas as telas de cadastro, atualiza contas/plano/centros/fornecedores/serviços de uma vez.
- **Pages/hubs:** `PageHeader` no topo de cada page; o hub `cadastros/page.tsx` vira **grade de cards clicáveis** (ícone/label) em vez de lista de links.

## 6. Testes e verificação
- **Unit:** `badgeStatusTitulo(status)` (mapa status→variante, incl. desconhecido→`neutro`).
- **Verificação:** `npm run lint && npm run typecheck && npm test && npm run build` (verde) + **checagem visual** desktop/mobile das telas financeiras (tabelas, badges de status, valores mono, dashboard, formulários, foco-visível).
- **Varredura:** sem resíduo `slate-*`/`bg-green-50`/`bg-red-50` nos arquivos migrados.
- Sem migration; RLS/cálculos/actions **inalterados**.

## 7. Riscos e observações
- **`ContasPagar`/`ContasReceber` são maiores e controlados** (estado de baixa/lançamento) — re-skin cuidadoso, preservando bindings e as ações (estorno com prompt, cobrança WhatsApp).
- **Barras do dashboard:** recolorir sem alterar os cálculos de `max`/proporção.
- **Consistência transitória:** NFS-e e o resto seguem no estilo antigo até V8.2c/d.
- **Reuso:** ganho de velocidade do `CadastroCrud` (um re-skin cobre 5 telas de cadastro).
