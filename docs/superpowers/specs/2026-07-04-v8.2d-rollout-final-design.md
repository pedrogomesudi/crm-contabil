# V8.2d — Rollout SALDO: Final (Atendimento, Integrações e Resto) — Design

**Data:** 2026-07-04
**Marco:** V8 — Layout e estética · **Sub-marco:** V8.2d (fatia final — fecha o rollout)
**Depende de:** V8.1 (tokens/primitivos), V8.2a/b/c (guia, primitivos ampliados, helpers, mapa de tokens).

## 1. Objetivo

Concluir o rollout SALDO: aplicar a identidade aos **19 arquivos restantes** (atendimento, config WhatsApp, Domínio/integrações, documentos, assinatura/contrato, usuários e sobras), deixando **toda a plataforma** na nova identidade — **zero resíduo de estilo antigo** em `src`. Só apresentação; actions/RLS/uploads/Clicksign inalterados.

## 2. Fronteiras (escopo)

**Dentro (19 arquivos):**
- Atendimento: `src/app/(app)/atendimento/{Inbox,page}.tsx`.
- Config WhatsApp: `src/app/(app)/configuracoes/whatsapp/{Formularios,page}.tsx`.
- Domínio/Integrações: `src/components/dominio/{AtualizarPelaReceita,ImportarEnderecos,PreviaImportacao,UploadDominio}.tsx`.
- Documentos: `src/components/documentos/{DocumentosSection,UploadDocumento,BotaoBaixar}.tsx`.
- Assinatura/Contrato: `src/components/assinatura/{EnviarAssinatura,StatusAssinatura}.tsx`, `src/components/contrato/GerarContrato.tsx`.
- Usuários: `src/app/(app)/usuarios/page.tsx`, `src/components/ConviteForm.tsx`.
- Sobras: `src/app/(app)/page.tsx` (resíduo do erro), `src/app/login/recuperar/page.tsx`, `src/components/CardResumo.tsx`.

**Fora:** nada — esta é a última fatia. Sem mudança de banco/lógica.

## 3. Linguagem (reuso)

Guia `docs/design/saldo-ui.md` + **mapa de tokens**: `slate-*`→tokens SALDO, botões primários→`Botao`/`verde`, sucesso→`verde/10`, erro→`negativo/10`, `amber`→manter (semântico), cards→`Card`/`Painel`, dados→`font-mono`, títulos→`font-display`. Regra invariável: **re-skin, nunca refuncionalizar**.

## 4. Toques específicos

- **Atendimento (`Inbox`)** — é uma UI de chat: lista de conversas em `Painel` (contato + prévia + **badge de não-lidas verde**); thread com **balões**: saída (`OUT`) fundo `verde/15` alinhado à direita, entrada (`IN`) fundo `creme`/branco à esquerda; caixa de resposta com `Input`/`Botao`. Mantém o polling e os handlers.
- **Usuários** — novo helper puro **`badgePapel(papel): "neutro" | "positivo" | "ia" | "atencao"`** (admin→`ia`/violeta, financeiro→`positivo`, contador→`neutro`, assistente→`atencao`) para a lista; convite (`ConviteForm`) com `Campo`/`Input`/`Select`.
- **Prévias de importação (Domínio)** e **listas de documentos** → `Painel` + tabelas no padrão; uploads/baixar via `Botao`.
- **Config WhatsApp** — o aviso do provedor não-oficial mantém o `amber`; credenciais em `Campo`/`Input` (password).

## 5. Testes e verificação
- **Unit:** `badgePapel(papel)` (mapa papel→variante, incl. desconhecido→`neutro`).
- **Verificação:** `npm run lint && npm run typecheck && npm test && npm run build` (verde) + **checagem visual** das telas (atendimento, usuários, integrações, documentos, config WhatsApp, recuperação de senha).
- **Varredura FINAL (gate do rollout):** `grep -rlE "slate-[0-9]|bg-green-50|bg-red-50|bg-blue-|border-slate" src/app src/components | grep -vE "ui/|marca/"` → **vazio** (zero resíduo em toda a plataforma). Exceção permitida: nenhuma.
- Sem migration; actions/RLS/uploads/Clicksign **inalterados**.

## 6. Riscos e observações
- **`CardResumo` legado:** re-skinnar (ou verificar se ainda tem consumidor; a home já usa `StatCard`). Se sem uso, pode ser removido; se em uso, re-skin.
- **Atendimento** é a tela com mais caráter visual (chat) — cuidar do contraste dos balões nos dois temas de leitura.
- **Regressões:** preservar uploads (documentos/Domínio), Clicksign (assinatura), convites (usuários), envio/polling (atendimento).
- **Fecho:** ao passar o gate de varredura, a **V8 (identidade + rollout) está completa**.
