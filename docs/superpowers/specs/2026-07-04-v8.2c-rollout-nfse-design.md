# V8.2c — Rollout SALDO: NFS-e — Design

**Data:** 2026-07-04
**Marco:** V8 — Layout e estética · **Sub-marco:** V8.2c (3ª fatia do rollout)
**Depende de:** V8.1 (tokens/primitivos), V8.2a (guia, primitivos ampliados, helpers, mapa de tokens), V8.2b (padrão de tabelas + badges de status).

## 1. Objetivo

Aplicar a identidade SALDO às telas de **NFS-e**, reusando os primitivos e o mapa de tokens já validados, com um helper novo para o status da nota. **Só apresentação** — mTLS/SEFIN, actions, cálculos e RLS inalterados.

## 2. Fronteiras (escopo)

**Dentro (13 arquivos):** `src/components/nfse/{EmitirNfseCliente,EmitenteConfig,EmissaoClienteSection,NotasFiscaisSection,LoteNfse,EmitirNfse,BaixarNfse,BaixarNotasZip,CancelarNfse}.tsx`, `src/app/(app)/nfse/lote/page.tsx`, `src/app/(app)/configuracoes/nfse/{page,Formularios}.tsx`. Helper novo `badgeStatusNfse`.

**Fora:** documentos/assinatura/contrato, atendimento, integrações, usuários → V8.2d. Sem mudança de banco/lógica.

## 3. Linguagem (reuso)

Guia `docs/design/saldo-ui.md` + **mapa de tokens** (das fatias anteriores): `slate-*`→tokens SALDO, botões primários→`Botao`/`verde`, sucesso→`verde/10`, erro→`negativo/10`, avisos `amber`→manter (semântico), cards→`Card`/`Painel`, **número da nota / CNPJ / valores → `font-mono`**, títulos→`font-display`. Regra invariável: **re-skin, nunca refuncionalizar**.

## 4. Toques específicos do NFS-e

- **`badgeStatusNfse(status: string): "positivo" | "neutro" | "negativo" | "atencao"`** (novo helper puro em `src/lib/ui/apresentacao.ts`): `autorizada`→`positivo`; `cancelada`→`neutro`; `rejeitada`/`erro`→`negativo`; `processando`/`pendente`/`enviada`→`atencao`; desconhecido→`neutro`. Case-insensitive.
- **Dados em mono:** `numero`, `cpf_cnpj`/CNPJ do tomador, `valor` (moeda) e códigos → `font-mono` com `tabular-nums`.
- **Ações:** emitir → `Botao primario`; cancelar → `Botao perigo` (mantendo confirmação); baixar PDF/XML/ZIP → `Botao secundario`/`fantasma`. Preservar handlers (emissão avulsa e por competência, cancelamento, ZIP).

## 5. Telas

- **`NotasFiscaisSection`** (lista de notas na ficha): `Painel` + tabela no padrão SALDO; status via `Badge`+`badgeStatusNfse`; número/valor em mono; botões de baixar/cancelar re-skinnados.
- **`EmissaoClienteSection` / `EmitirNfseCliente` / `EmitirNfse`:** formulários de emissão (avulsa e por honorário) com `Campo`/`Input`/`Select`/`Botao`; feedback de sucesso/erro semântico. **Não** alterar o fluxo de emissão avulsa.
- **`LoteNfse` + `nfse/lote/page`:** lista/seleção de clientes para emissão em lote em `Painel`; `PageHeader`; status/resultado com `Badge`.
- **`EmitenteConfig` + `configuracoes/nfse/{page,Formularios}`:** `PageHeader` + `Card` para as seções (dados do emitente, certificado); campos com `Campo`/`Input`/`Select` (o `Formularios` tem um `inputCls` local → substituir pelo padrão SALDO); botões via `Botao`.
- **`BaixarNfse` / `BaixarNotasZip` / `CancelarNfse`:** botões re-skinnados (baixar=`secundario`, cancelar=`perigo`), mensagens semânticas; comportamento preservado.

## 6. Testes e verificação
- **Unit:** `badgeStatusNfse(status)` (mapa status→variante, incl. case-insensitive e desconhecido→`neutro`).
- **Verificação:** `npm run lint && npm run typecheck && npm test && npm run build` (verde) + **checagem visual** (listas de notas, emissão, config, lote) desktop/mobile.
- **Regressão crítica:** confirmar que a **emissão avulsa** e a emissão em lote continuam funcionando (só re-skin).
- **Varredura:** sem resíduo `slate-*`/`bg-green-50`/`bg-red-50` nos arquivos migrados.
- Sem migration; RLS/actions/mTLS **inalterados**.

## 7. Riscos e observações
- **Emissão avulsa (restrição do usuário):** o fluxo que já funciona não pode quebrar — re-skin cuidadoso, preservar `name`/`value`/`onChange`/actions.
- **`Formularios` de config tem `inputCls` local:** trocar pela classe SALDO (ou importar a de `ui/Campo`) para consistência.
- **Consistência transitória:** documentos/assinatura e o restante seguem no estilo antigo até a V8.2d.
