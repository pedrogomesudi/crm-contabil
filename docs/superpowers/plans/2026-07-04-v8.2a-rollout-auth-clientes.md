# V8.2a — Rollout SALDO (Auth + Clientes) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fixar a linguagem de design SALDO, ampliar os primitivos de UI e recomponentizar as áreas de Autenticação e Clientes (estilo Apple), como referência do rollout.

**Architecture:** Guia de design + primitivos novos (`src/components/ui/`) + helpers puros. Recomponentização re-skinna as telas usando os primitivos e o mapa de tokens, **preservando toda a lógica** (permissões, actions, RLS, formulários controlados).

**Tech Stack:** Next.js 16, Tailwind 4 (tokens SALDO da V8.1), TypeScript, Vitest (jsdom + `renderToStaticMarkup`). Sem banco, sem novas dependências.

## Global Constraints

- **Só visual:** nunca alterar `name`/`value`/`onChange`/action/props funcionais nem RLS. Preservar `aria-*`, `role="alert"`, labels.
- **Mapa de tokens (aplicar em toda re-skin):** `text-slate-900→text-texto`; `text-slate-700/600→text-cinza`; `text-slate-500→text-cinza-claro`; `border-slate-*→border-linha`; `bg-slate-100/50→bg-creme`; botão primário `bg-slate-900 text-white`→`<Botao variante="primario">`; botão secundário `border-slate-300`→`<Botao variante="secundario">`; sucesso `bg-green-50 text-green-700`→`bg-verde/10 text-verde`; erro `bg-red-50 text-red-700`→`bg-negativo/10 text-negativo`; `rounded`→`rounded-lg`; cartões→`Card`/`Painel` (`rounded-2xl`); inputs→`Campo`+`Input`; dados (CNPJ/R$)→`font-mono`.
- **Fontes/tokens:** já existem (V8.1): `font-display/sans/mono`, `verde/tinta/creme/linha/cinza/cinza-claro/texto/violeta/negativo`.
- **Antes de commitar:** `npm run lint && npm run typecheck && npm test && npm run build`.
- **Alias** `@/*` → `./src/*`.

```ts
// 2 iniciais de um nome (composto → 1ª de cada uma das 2 primeiras palavras; 1 palavra → 2 letras).
export function iniciais(nome: string): string {
  const partes = String(nome ?? "").trim().split(/\s+/).filter(Boolean);
  if (partes.length === 0) return "?";
  if (partes.length === 1) return partes[0]!.slice(0, 2).toUpperCase();
  return (partes[0]![0]! + partes[1]![0]!).toUpperCase();
}

// Regime tributário → variante de cor do Badge.
export function badgeRegime(regime: string | null): "positivo" | "ia" | "neutro" | "atencao" {
  const r = (regime ?? "").toLowerCase();
  if (r.includes("simples")) return "positivo";
  if (r.includes("presumido")) return "ia";
  if (r.includes("mei")) return "atencao";
  return "neutro";
}
```
